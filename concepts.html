Key Concepts

These terms help explain how browsers, servers, and secure communication work together.

Client-server model

The browser sends a request and the server sends a response. This request-response model is one of the main ideas behind HTTP.

Stateless protocol

HTTP does not remember earlier requests unless extra tools are used. This makes the protocol simpler but also creates the need for sessions and cookies.

TCP

Used by earlier HTTP versions for reliable delivery. TCP helps make sure data arrives correctly and in order.

QUIC

Used by HTTP/3 to improve speed and reduce delay. QUIC is part of the reason HTTP/3 performs better on modern connections.
