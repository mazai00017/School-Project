Key Concepts

Terms That Explain the Protocol Stack

These concepts connect the research pages and help explain the technical vocabulary behind web communication.

Key concepts and definitions

Terms used throughout the website
TermDefinitionWhy it matters
ClientThe device or software that starts the request, usually a browser.Explains why web communication begins with the user agent.
ServerThe system that receives a request and returns a response.Shows where web resources and application logic are delivered from.
RequestA message sent by the client asking for a resource or action.Forms the basic unit of interaction in HTTP.
ResponseA message sent by the server after processing the request.Includes content, status codes, and headers.
StatelessnessHTTP does not automatically remember previous requests.Explains why cookies, sessions, and tokens are needed for logins.
TLSA security protocol that encrypts and authenticates communication.Provides the protection that makes HTTPS useful.
Certificate AuthorityAn organization trusted to issue digital certificates.Helps browsers verify that a site is who it claims to be.
QUICA modern transport protocol used by HTTP/3.Improves connection setup, security integration, and stream behavior.
CacheA stored copy of a resource used to avoid unnecessary transfers.Improves speed and reduces server load.
Status codeA three-digit number explaining the result of a request.Lets browsers and developers understand success, redirects, and errors.

Methods

Common HTTP methods include GET for retrieving resources and POST for sending data to be processed by the server.

Headers

Headers carry metadata such as content type, cache rules, cookies, accepted languages, and authentication tokens.

Status codes

Status codes such as 200, 301, 404, and 500 quickly describe whether a request succeeded, redirected, failed, or caused a server error.

Concept map

HTTP defines the meaning of web messages. TLS protects those messages. QUIC changes how modern HTTP can move across a network. Together, they explain why the modern web can be interactive, secure, and fast.
