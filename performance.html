HTTP/3

HTTP/2, HTTP/3, and QUIC

Modern HTTP versions improve the way browsers load many resources while keeping the same core request-response meaning.

Why performance changed after HTTP/1.1

Older web pages often opened several connections to load many resources. HTTP/2 improved this by allowing multiple streams to share a single connection through binary framing and multiplexing. However, because HTTP/2 commonly uses TCP, one lost packet can still delay unrelated streams at the transport layer.

HTTP/3 addresses this by using QUIC. QUIC provides stream-level behavior over UDP, integrates TLS 1.3, and supports faster connection setup. The goal is not only raw speed, but smoother performance on real networks where loss and switching connections are common.

HTTP/3 design goals

1

Reduce connection setup delay.

2

Prevent one blocked stream from freezing all active streams.

3

Keep modern encryption built into the transport design.
